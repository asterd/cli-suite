# Milestone 9 Status Report

## What was implemented
- Added the `axt-test` binary crate and wired it into the workspace.
- Defined the `TestFrontend` trait, normalized test models, and `axt.test.v1` JSON envelope plus JSONL record schemas.
- Implemented framework detection and command mapping for jest, vitest, pytest, cargo test, go test, bun test, and deno test.
- Implemented parsers for normalized line records, Jest-style JSON documents, pytest-json-report-style documents, Go `-json` events, and stable `cargo test` text output.
- Added `list-frameworks`, `--framework`, `--filter`, `--files`, `--changed`, `--changed-since`, `--single`, `--bail`, `--workers`, `--top-failures`, `--include-output`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--plain`, `--print-schema`, and `--list-errors`.
- Integrated `--changed` and `--changed-since` with `axt-git`; added `StatusCache::changed_paths()` to expose repository-relative dirty paths.
- Added config override detection for `axt-test.toml`, `[tool.axt-test]`, and `package.json#axt-test.framework`.
- Added live JSONL streaming for line-oriented framework output; case records are flushed while the framework process is still running, with suite and summary records emitted at the end.
- Added committed fixture project markers for all seven supported frameworks and integration tests with fake local toolchains so CI does not require Node, Python pytest, Go, Bun, or Deno installations.
- Added JSON/JSONL schema validation tests and committed JSONL plus ACF snapshots for every supported framework.
- Added `docs/commands/test.md` with CLI, output, changed-file behavior, error codes, cross-platform notes, and per-framework mapping.

## What was not implemented and why
- Real framework reporter integration is partial. The parsers support native Go JSON, stable Cargo text fallback, Jest-like JSON documents, pytest-json-report-like JSON documents, and normalized JSON line records; bundled reporters for frameworks without native machine output were not added.

## Deviations from the spec
- `cargo test` uses stable text parsing instead of `--format json -Z unstable-options`, because the specified libtest JSON path requires unstable Rust flags. This is documented in `docs/commands/test.md`.
- Jest and Vitest invoke `npm test --` instead of `npx jest` or `npx vitest`, to avoid any chance of network behavior and to respect the project script.
- Fixture tests use fake local toolchain commands in `PATH` rather than installing every framework, keeping the workspace offline and deterministic.

## How to run tests
```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --tag v0.6.0-rc1 --output-format=json
cargo publish -p axt-test --dry-run --allow-dirty
cargo install --path crates/axt-test --locked --debug --root /tmp/axt-test-install-check
/tmp/axt-test-install-check/bin/axt-test --version
```
