# Milestone 2 Status Report

## What was implemented
- Implemented `ax-fs` directory walking on `ignore::WalkBuilder`, honoring `.ignore` and `.gitignore` by default with `WalkOptions::no_ignore` to disable ignore handling.
- Added deterministic `EntryMetadata` extraction in `ax-fs`: relative path, kind, size, mtime, executable bit, language, MIME, content kind, encoding, newline style, generated-likely flag, and optional BLAKE3 hash.
- Added `ax-fs` symlink policy support: symlinks are not followed by default, and followed directory loops are guarded by filesystem identity tracking.
- Added language detection through an extension table with `infer` fallback, MIME detection through `mime_guess` with `infer` fallback, and generated-file heuristics for known generated directories, header markers, and minified JavaScript.
- Implemented `ax-git` repository discovery with graceful `Ok(None)` for paths outside a Git repository.
- Added `ax-git` wrappers for cached status, per-path status lookup, current branch, dirty counts, and changed paths between revisions.
- Added staged-status support in `ax-git` by comparing the index against `HEAD`, then merging worktree and untracked status results for monorepo-friendly cached lookups.
- Added committed fixtures under `fixtures/fs-small` and `fixtures/fs-with-git` plus unit tests for deterministic filesystem metadata, ignore behavior, hashing/classification, repository absence, status/branch/dirty counts, and revision diffs.

## What was not implemented and why
- No `ax-peek` integration was implemented because wiring `ax-fs` and `ax-git` into command output belongs to Milestone 3.
- No public `ax.*.v1` schema changes were made because M2 only adds internal shared crates.
- No SHA-256 hashing was implemented because the spec reserves it for a future v2 if requested.
- Full CI on Linux/macOS/Windows was not observed from this local session; only local workspace quality gates were run.

## Deviations from the spec
- No intentional spec deviations.
- The `fixtures/fs-with-git` tests initialize a temporary Git repository from committed fixture files instead of committing nested `.git` metadata, keeping the fixture portable while still exercising Git-backed behavior.
