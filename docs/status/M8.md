# Milestone 8 Status Report

## What was implemented
- Added the `axt-port` binary crate and wired it into the workspace as the Phase 5 command.
- Implemented `axt-port list`, `who <PORT>...`, `free <PORT>...`, and `watch <PORT> --timeout <DURATION>`.
- Added shared filters for protocol, loopback inclusion, listening-only mode, bind host, owner, and PID.
- Implemented local socket discovery on Linux via `/proc`, on macOS via `lsof`, and on Windows via `netstat -ano`.
- Implemented process enrichment with PID, parent PID, process name, command, owner, memory, bind addresses, and best-effort cwd where available.
- Implemented `free` safety controls: `--dry-run`, `--confirm`, PID 1 refusal, self refusal, parent refusal unless `--force-self`, signal selection, grace-period escalation, and direct-child `--tree` handling.
- Added all shared output modes: human, `--plain`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--print-schema`, and `--list-errors`.
- Added `schemas/axt.port.v1.schema.json` and integration tests that validate JSON output against it.
- Added fixture-process tests for known-port `who`, `free --dry-run`, actual `free --signal kill`, `watch --timeout`, JSONL, ACF, schema printing, and remote-host syntax refusal.
- Added `docs/commands/port.md` with CLI, output, safety, error, and cross-platform notes.
- Recovery update: `watch` now reports an already-free port as `freed=true` without timing out, JSONL record schemas are committed and validated in tests, fixture-based JSONL/ACF snapshots are committed, and Windows process metadata lookup now uses `Win32_Process`.
- Recovery update: Windows `free --tree` now passes `/T` to `taskkill`; Unix tree behavior remains process-parent based.

## What was not implemented and why
- Full native Windows Job Object signaling was not implemented. Existing processes cannot be retroactively contained in a new Job Object, so `taskkill /T` is used for Windows process-tree termination; local GitHub CI execution still must be observed outside this local session.
- macOS `cwd` and `started` are best-effort and currently remain `null` when unavailable from unprivileged local inspection, matching the spec's graceful-degradation rule.

## Deviations from the spec
- `sysinfo`, `netstat2`, `nix`, and `windows` crate dependencies were not added. The implementation uses existing dependencies plus local OS facilities (`/proc`, `lsof`, `netstat`, `kill`, `taskkill`) to avoid new network dependency resolution and keep the change surgical.
- `--timeout`, `--tree`, and `--force-self` were implemented even though they are not listed in §11.3.3 because §11.3.7 and §11.3.8 require those behaviors.

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --tag v0.5.0-rc1 --output-format=json
cargo publish -p axt-port --dry-run --allow-dirty
cargo install --path crates/axt-port --locked --debug --root /tmp/axt-port-install-check
/tmp/axt-port-install-check/bin/axt-port --version
```
