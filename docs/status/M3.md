# Milestone 3 Status Report

## What was implemented
- Replaced the `ax-peek` stub with the section 9 module layout: `cli`, `command`, `collect`, `model`, `render`, and `error`.
- Wired the `ax-peek` CLI surface for paths, depth, file/dir filters, hidden/ignore handling, git toggles, changed filters, type/language filters, hashing, sorting, symlink/cross-filesystem flags, shared output modes, limits, and standard shared flags.
- Implemented `PeekData`, `Entry`, and `Summary` models with `serde` and `schemars` support for the `ax.peek.v1` JSON payload.
- Implemented collection through `ax-fs` and `ax-git`, including deterministic walking, metadata mapping, BLAKE3 opt-in hashes, git status lookup, `--changed`, and `--changed-since <REF>`.
- Implemented human/plain, JSON, JSON data, JSONL, and ACF agent rendering, including summary-first line output and explicit truncation warnings.
- Replaced the stub schema with `schemas/ax.peek.v1.schema.json` for the MVP payload and added schema validation in tests.
- Rewrote `crates/ax-peek/tests/modes.rs` to cover mode output, JSON schema validation, JSONL record contract, ACF contract, truncation, depth, hashing, summary-only, missing paths, conflicting flags, and git changed filters.
- Rewrote `docs/commands/peek.md` to document the implemented command, flags, output records, agent keys, errors, and cross-platform notes.
- Added recovery coverage for `--print-schema agent`, plain mode, empty directories, depth 0/10, byte-limit truncation metadata, permission-denied warnings, symlink-loop warnings, and submodule-like directories.
- Extended `ax-fs` with non-fatal warning collection so `ax-peek` can continue past permission-denied subtrees and followed symlink loops.
- Added optional shared schema formats in `ax-core` so `--print-schema [human|json|jsonl|agent]` parses while preserving bare `--print-schema`.

## What was not implemented and why
- Full GitHub draft release publishing was not performed from this local session; local `cargo-dist` generation and plan dry-runs pass, but actual draft publishing requires a tag and GitHub credentials.
- CI on Linux/macOS/Windows was not observed from this local session.
- True non-UTF-8 path rendering remains limited by the current UTF-8 path model; non-UTF-8 child paths are reported as warnings rather than rendered as entries.
- A real Git submodule fixture was not added; the behavior is covered with a submodule-like directory containing a `.git` file.

## Deviations from the spec
- Directory entries use `bytes: 0` in command output, matching the section 9 examples, even though filesystem metadata may report platform-specific directory sizes.
- `--changed-since <REF>` compares `<REF>` to `HEAD`; it does not include unstaged working-tree changes in that comparison.
- `--type` filters are pragmatic MIME/path/language heuristics rather than a formal content taxonomy.
- The command supports multiple input paths internally, but paths are rendered relative to each individual scan root; no root-prefix disambiguation is added for same-named files across multiple roots.
- `ax.peek.v1` was promoted from the M0/M1 stub payload to the real M3 payload without a major schema version bump, because Milestone 3 is the first non-stub `ax-peek` contract.
- `--print-schema human|jsonl|agent` emits compact textual contract descriptions; only `--print-schema json` emits a JSON Schema file.

## How to run tests
```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --output-format=json
```

Both commands were run locally. The first `dist generate --check` attempt failed under the sandbox because `cargo metadata` needed missing registry artifacts; after approved network access for `rtk dist`, `dist generate --check` passed. `dist plan --output-format=json` also passed.
