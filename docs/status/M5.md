# Milestone 5 Status Report

## What was implemented
- Added the `axt-run` binary crate and wired it into the workspace as the Phase 2 command.
- Implemented command execution with `tokio::process::Command`, cwd selection, env/env-file support, optional shell mode, stream capture, timeout handling, and correct command exit-code mapping.
- Added saved run artifacts under `.axt/runs/<name>/`: `meta.json`, `stdout.log`, `stderr.log`, `changed.json`, and `summary.agent.acf`.
- Implemented `axt-run show [<NAME>|last]`, `axt-run list`, and `axt-run clean [--older-than <DURATION>]`.
- Implemented file-change detection for created, modified, and deleted files using size, mtime, inode where available, and optional BLAKE3 hashing via `--hash`.
- Added all shared output modes for `axt-run`: human, `--plain`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--print-schema`, and `--list-errors`.
- Added `schemas/axt.run.v1.schema.json` and integration tests that validate JSON output against it.
- Replaced the placeholder `docs/commands/run.md` with CLI, output, storage, retention, and cross-platform documentation.
- Added cross-platform integration coverage using a compiled Rust helper for deterministic command output, environment passing, timeouts, file changes, saved runs, listing, showing, and cleaning.
- Recovery update: added `--no-watch-files`, schema coverage for subcommands, committed `insta` snapshots for human/JSONL/agent output, documented all `axt-run` ACF prefixes and error codes, and implemented Windows Job Object timeout cleanup.

## What was not implemented and why
- `--capture auto` currently behaves like `always`, because `axt-run` never inherits the child tty and reliable summaries require piped streams.

## Deviations from the spec
- Added `--hash` to the CLI because §10.5 specifies it in implementation notes even though §10.2 omits it from the option list.
- Shared output flags for subcommands are accepted before the subcommand, for example `axt-run --json-data list`; this matches the workspace's existing top-level shared-flag pattern.
- `--capture auto` currently behaves like `always`; this preserves reliable structured summaries because `axt-run` never inherits the child tty.

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --tag v0.2.0-rc1 --output-format=json
cargo publish -p axt-run --dry-run --allow-dirty
cargo install --path crates/axt-run --locked --debug --root /tmp/axt-run-install-check
/tmp/axt-run-install-check/bin/axt-run --version
```
