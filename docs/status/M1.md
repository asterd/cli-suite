# Milestone 1 Status Report

## What was implemented
- Added `ax-core` foundations: `ErrorCode`, exit-code mapping, `STANDARD_ERROR_CATALOG`, retryability markers, and serde string serialization for error codes.
- Added shared CLI primitives in `ax-core`: `OutputMode`, clap-backed output mode conflict detection, output limit flags, `--print-schema`, and `--list-errors` support through `CommonArgs`.
- Added `Clock`, `SystemClock`, and feature/test-gated `FixedClock`, plus `CommandContext`, UTF-8 current-directory handling, and color choice resolution for `NO_COLOR`, `CLICOLOR_FORCE`, and `FORCE_COLOR`.
- Added `ax-output` foundations: `Renderable`, `RenderContext`, `JsonEnvelope`, `OutputDiagnostic`, typed output errors, style helpers, and stdout/stderr stream helpers using `anstream`/`anstyle`.
- Added `AgentNdjsonWriter`, with minified NDJSON output, record and byte truncation, truncation warning records, and strict truncation errors.
- Rewired the existing `ax-peek` stub to use `ax-core` and `ax-output` while preserving the existing stub data payloads for human, JSON, JSON data, and agent modes.
- Added a tiny published schema for the M1 `ax.peek.v1` stub output at `schemas/ax.peek.v1.schema.json`.
- Added unit tests for `ax-core` and `ax-output`, plus `ax-peek` integration tests with inline snapshots, JSON schema validation, NDJSON validation, catalog listing checks, and clap conflict checks.

## What was not implemented and why
- No real `ax-peek` filesystem, git, language, or traversal behavior was implemented because that belongs to Milestone 3.
- No `ax-fs`, `ax-git`, or later binary behavior was implemented because this session is limited to Milestone 1.
- Full cross-platform CI was not observed from this local session; only local quality gates were run.

## Deviations from the spec
- No intentional spec deviations.
- The M1 "hello world" binary was implemented by rewiring the existing `ax-peek` stub, not by adding a new binary, to respect the project rule forbidding binaries beyond the six named commands.

## How to run tests
```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --output-format=json
```

These were attempted in this session, but both failed before planning because `cargo metadata` tried to download uncached crates and the sandbox could not resolve `static.crates.io`.
