# Milestone 4 Status Report

## What was implemented
- Fixed the online clippy failure in `axt-peek` agent rendering by replacing manual loop counters with iterator-derived record indexes.
- Bumped the workspace and releasable crates to `0.1.0-rc1` so `dist plan --tag v0.1.0-rc1` resolves the intended prerelease.
- Renamed publishable internal package names to `axt-core`, `axt-output`, `axt-fs`, and `axt-git` while preserving the Rust crate names `axt_core`, `axt_output`, `axt_fs`, and `axt_git`.
- Added versioned path dependencies required for publishing `axt-peek` to crates.io.
- Raised the workspace Rust version to `1.82` and updated `gix` to `0.77` to clear RustSec advisories from the release dependency graph.
- Updated `CHANGELOG.md` with the `0.1.0-rc1` release notes used by cargo-dist.
- Replaced the placeholder `docs/release.md` with a release runbook covering preflight, publish order, tagging, Scoop generation, smoke tests, and recovery.
- Added `scripts/release/scoop-manifest.py` to generate `bucket/axt-peek.json` from a cargo-dist manifest and the Windows archive SHA-256 file.
- Added a focused unittest for the Scoop manifest generator.
- Added CI advisory checks with `cargo-audit` and `cargo-deny`.
- Added a release follow-up workflow that downloads release artifacts, creates GitHub Artifact Attestations, generates and uploads a CycloneDX SBOM, and opens a Scoop bucket PR from the generated manifest.
- Added Python bytecode ignores and removed generated `__pycache__` artifacts from the working tree.

## What was not implemented and why
- `v0.1.0-rc1` was not tagged or pushed; that requires maintainer-controlled signing and remote release credentials.
- GitHub Release, Homebrew tap update, Scoop bucket PR, artifact attestation, SBOM upload, and installer smoke tests were not run; they require the pushed tag, repository secrets, and external clean machines or VMs.
- `cargo publish --dry-run` for `axt-output` and `axt-peek` cannot complete before their unpublished internal prerelease dependencies exist on crates.io. Independent internal crates were dry-run validated, and the runbook documents the real publish order.
- `v0.1.0` was not tagged or promoted; the release candidate has not yet been externally smoke-tested.

## Deviations from the spec
- Public crates.io package names now use the reserved `axt-*` namespace instead of the earlier `ax-foundation-*` fallback.
- The workspace `rust-version` is now `1.82` instead of the original `1.78`; this was required to move `gix` to a non-vulnerable dependency line for the release security gates.
- Scoop, artifact attestation, and SBOM publication are wired through `.github/workflows/release-followup.yml` instead of hand-editing the generated cargo-dist `.github/workflows/release.yml`.

## How to run tests
```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
python3 -m unittest scripts/release/test_scoop_manifest.py
cargo audit
cargo deny check advisories
```

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --tag v0.1.0-rc1 --output-format=json
cargo publish -p axt-core --dry-run --allow-dirty
cargo publish -p axt-fs --dry-run --allow-dirty
cargo publish -p axt-git --dry-run --allow-dirty
cargo install --path crates/axt-peek --locked --debug --root /tmp/axt-peek-install-check
/tmp/axt-peek-install-check/bin/axt-peek --version
```

For dependent crate publishing, commit first, then publish in the documented order from `docs/release.md`.
