# Milestone 6 Status Report

## What was implemented
- Added the `axt-doc` binary crate and wired it into the workspace as the Phase 3 command.
- Implemented `axt-doc which <CMD>` with all PATH matches, primary resolution, version-manager attribution, and timeout-bound `<CMD> --version` probing.
- Implemented `axt-doc path` with PATH entry listing, duplicate detection, missing directory detection, broken symlink detection, and ordering issue reporting.
- Implemented `axt-doc env` with environment variable listing, secret-like variable detection, redaction by default, suspicious empty variable reporting, and `--show-secrets` with a stderr warning.
- Implemented `axt-doc all <CMD>` as a combined response for `which`, PATH, and env diagnostics.
- Added all shared output modes for `axt-doc`: human, `--plain`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--print-schema`, and `--list-errors`.
- Added `schemas/axt.doc.v1.schema.json` and integration tests that validate JSON output against it.
- Replaced the placeholder `docs/commands/doc.md` with CLI, output, secret-handling, and cross-platform documentation.
- Added best-effort manager detection by path patterns and timeout-bound local manager probes or env-root detection for Homebrew, mise, asdf, rustup, cargo bin, pyenv, rbenv, volta, nvm-shim, Scoop, Chocolatey, and winget.

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --tag v0.3.0-rc1 --output-format=json
cargo publish -p axt-doc --dry-run --allow-dirty
cargo install --path crates/axt-doc --locked --debug --root /tmp/axt-doc-install-check
/tmp/axt-doc-install-check/bin/axt-doc --version
```
