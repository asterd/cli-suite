# Milestone 0 Status Report

## What was implemented
- Added the Rust workspace root `Cargo.toml` with the Phase 1 members from `docs/spec.md` section 7.
- Added empty internal library crates: `axt-core`, `axt-output`, `axt-fs`, and `axt-git`.
- Added the `axt-peek` binary crate as a Milestone 0 stub with placeholder support for human, `--plain`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--print-schema`, and `--list-errors`.
- Added the `xtask` stub crate.
- Added top-level project files: `README.md`, `CHANGELOG.md`, `CONTRIBUTING.md`, `SECURITY.md`, `LICENSE-MIT`, and `LICENSE-APACHE`.
- Added documentation scaffold files under `docs/` and command docs under `docs/commands/`.
- Added fixture and schema directories with placeholder files so the repository structure is present.
- Added `.github/workflows/ci.yml` with Linux, macOS, and Windows stable Rust jobs running fmt, clippy, and test.
- Ran `dist init --yes --ci=github --installer shell --installer powershell --installer homebrew`, added `dist-workspace.toml`, and generated `.github/workflows/release.yml`.
- Configured cargo-dist targets to the tier-1 target list from `docs/spec.md` section 12.2.

## What was not implemented and why
- No `axt-peek` filesystem, git, language-detection, or rendering logic was implemented because that belongs to later milestones.
- No `axt-run`, `axt-doc`, `axt-drift`, `axt-port`, or `axt-test` crates were added because Milestone 0 only asks for the Phase 1 crate stubs.
- A real GitHub Actions run was not observed because this local directory is not currently a Git repository and no push occurred.
- A draft GitHub Release from tagging `v0.0.1` was not produced because that requires a GitHub repository, tag push, and release workflow execution.

## Deviations from the spec
- `cargo install cargo-dist` installed a `dist` executable for cargo-dist v0.31.0, but it did not register a `cargo dist` subcommand in this environment. The release files were generated with the equivalent `dist init` / `dist generate` commands from the installed cargo-dist tool.
- The workspace repository and homepage were set to `https://github.com/ddurzo/axt` so cargo-dist can generate release URLs. Confirm or replace this before publishing.
- The `axt-peek` stub includes minimal placeholder output-mode handling to satisfy the project hard rule that stub commands support the required modes; it does not implement real command behavior.

## How to run tests
```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

## How to run the release pipeline locally
```bash
dist generate --check
dist plan --output-format=json
```
