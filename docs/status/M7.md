# Milestone 7 Status Report

## What was implemented
- Added the `axt-drift` binary crate and wired it into the workspace as the Phase 4 command.
- Implemented `axt-drift mark [--name <NAME>] [--hash]`, defaulting to the `default` mark.
- Implemented JSONL snapshot storage at `.axt/drift/<NAME>.jsonl` with relative path, size, mtime, and optional BLAKE3 hash records.
- Implemented `axt-drift diff [--since <NAME>] [--hash]` with created, modified, and deleted files sorted by absolute size delta.
- Implemented `axt-drift run [--name <NAME>] [--hash] -- <CMD>` to snapshot before execution, run the command locally, and report filesystem drift after execution.
- Implemented `axt-drift list` and `axt-drift reset` for stored drift marks.
- Added all shared output modes for `axt-drift`: human, `--plain`, `--json`, `--json-data`, `--jsonl`, `--agent`, `--print-schema`, and `--list-errors`.
- Added `schemas/axt.drift.v1.schema.json` and integration tests that validate JSON output against it.
- Replaced the placeholder `docs/commands/drift.md` with CLI, output, storage, hash-mode, and cross-platform documentation.

## What was not implemented and why
- Nothing known for the Milestone 7 spec boundary.

## Deviations from the spec
- `--hash` was added as an explicit opt-in flag for `mark`, `diff`, and `run` because §11.2 says hashes are optional and the cross-platform matrix says hash mode is opt-in.
- The default mark name is `default`, resolving the unspecified `--name` / `--since` default while preserving the specified `.axt/drift/<NAME>.jsonl` storage shape.

```bash
cargo publish -p axt-drift --dry-run --allow-dirty
cargo install --path crates/axt-drift --locked --debug --root /tmp/axt-drift-install-check
/tmp/axt-drift-install-check/bin/axt-drift --version
```
