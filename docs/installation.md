# Installation

`axt` commands are independent binaries. The canonical names are:

- `axt-peek`
- `axt-run`
- `axt-doc`
- `axt-drift`
- `axt-port`
- `axt-test`

## Supported Channels

| Channel | Linux | macOS | Windows | Status |
|---|---:|---:|---:|---|
| Cargo | Yes | Yes | Yes | Local checkout works now; crates.io publish is release-gated. |
| GitHub Releases | Yes | Yes | Yes | Generated by `cargo-dist` after release tags. |
| Shell installer | Yes | Yes | No | Generated by `cargo-dist`. |
| PowerShell installer | No | No | Yes | Generated by `cargo-dist`. |
| Homebrew | No | Yes | No | Generated by `cargo-dist` for macOS. |
| Scoop | No | No | Yes | Manifest tooling lives in `scripts/release/`. |

## Local Checkout

Install the complete suite:

```bash
python3 scripts/install-local.py --command all
```

Install one command with the helper:

```bash
python3 scripts/install-local.py --command peek
python3 scripts/install-local.py --command run
python3 scripts/install-local.py --command doc
python3 scripts/install-local.py --command drift
python3 scripts/install-local.py --command port
python3 scripts/install-local.py --command test
```

Install one command directly with Cargo:

```bash
cargo install --path crates/axt-peek --locked
```

Install with opt-in aliases:

```bash
cargo install --path crates/axt-peek --locked --features aliases
```

Repeat for the command crate you need.

## Aliases

Aliases are opt-in to avoid collisions with existing commands.

| Canonical | `ax-*` alias | Short alias |
|---|---|---|
| `axt-peek` | `ax-peek` | `peek` |
| `axt-run` | `ax-run` | `run` |
| `axt-doc` | `ax-doc` | `doc` |
| `axt-drift` | `ax-drift` | `drift` |
| `axt-port` | `ax-port` | `port` |
| `axt-test` | `ax-test` | `test` |

Use canonical names in scripts. Use aliases only for interactive convenience or controlled agent environments.

## Release Install Commands

The final URLs and package names are release-owner controlled. After the first public release, the intended commands are:

| System | Command |
|---|---|
| Linux | `curl --proto '=https' --tlsv1.2 -LsSf <release-install-url> | sh` |
| macOS | `brew install <org>/axt/axt-peek` |
| Windows | `irm <release-install-url> | iex` or `scoop install axt-peek` |
| Any Rust host | `cargo install axt-peek --locked` |

Repeat for each command package.

## Verification

After installation:

```bash
axt-peek --version
axt-peek --help
axt-peek --print-schema json
axt-peek --list-errors
```

For alias installs:

```bash
ax-peek --version
peek --version
```

## Agent Skill Install

Install the suite skill into the current project for Codex and Claude Code:

```bash
python3 scripts/agent/install-skills.py --agent both --scope project --skill axt-suite
```

Install every command-specific skill globally:

```bash
python3 scripts/agent/install-skills.py --agent both --scope user --skill all
```

Codex project skills are copied to `.codex/skills/`. Claude Code project skills
are copied to `.claude/skills/`. Restart the agent after installation.
